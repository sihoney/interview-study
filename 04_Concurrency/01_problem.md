# 01. Concurrency — Problem Definition

---

## 🧨 장애 시나리오

이커머스 시스템에서  
동시에 주문 요청이 들어오면 재고가 음수가 되거나  
결제 금액이 틀어지는 문제가 발생한다.

간헐적으로만 발생하므로  
디버깅이 매우 어렵고, 재현도 힘들다.

---

## 🔎 관찰된 증상

- 동일 상품 재고가 음수
- 결제 금액 불일치
- 로그에는 정상처럼 보임
- 서버 스레드 수 증가 시 오류 증가

---

## ❓ 핵심 질문

1. 동시성이 왜 위험한가?
2. Java 스레드는 어떻게 동작하는가?
3. Race Condition은 무엇인가?
4. synchronized, volatile, Atomic은 어떤 차이가 있는가?

---

## 🧠 학습 목표

이 과정을 통해 다음을 설명할 수 있어야 한다.

- 스레드 생성과 실행 구조
- Race Condition 발생 원리
- Java 동기화 기법의 차이와 적용 기준
- 성능과 안정성의 트레이드오프

---

## 🧭 앞으로의 흐름

1. 멀티스레드 이론 정리
2. Race Condition 재현
3. 동기화 전략 설계
4. 성능 비교 실험
5. 면접 답변 완성
