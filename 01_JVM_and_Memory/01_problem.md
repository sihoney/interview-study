# 01. JVM & Memory — Problem Definition

## 🧨 장애 시나리오

다음과 같은 상황이 발생했다.

> 서비스 트래픽이 증가한 이후  
> 서버가 점점 느려지더니 결국 응답하지 않는다.  
> CPU 사용률은 정상이지만, 메모리는 충분히 남아 있는 상태다.  
> 로그에는 Full GC가 반복적으로 발생하고 있다.

---

## 🔎 관찰된 증상

- 요청 처리 지연
- 간헐적인 서버 응답 없음
- GC 로그 폭증
- Heap 메모리는 아직 여유 있음
- CPU는 과부하 아님

---

## ❓ 핵심 질문

이 장애를 이해하고 해결하기 위해 다음 질문에 답해야 한다.

1. JVM은 정확히 무엇을 관리하는가?
2. JVM 메모리는 어떻게 구성되어 있는가?
3. 객체는 어디에서 생성되고 언제 제거되는가?
4. GC는 언제 작동하며, 왜 서버를 멈추는가?
5. String, static, main 메서드는 메모리에서 어떻게 관리되는가?
6. 잘못된 객체 생성 패턴은 어떤 장애를 만드는가?

---

## 🧠 학습 목표

이 과정을 통해 다음을 설명할 수 있어야 한다.

- JVM 구조를 그림 없이 설명할 수 있다.
- 객체 생애 주기를 Heap, Stack 관점에서 설명할 수 있다.
- GC 동작 원리와 서버 장애의 관계를 설명할 수 있다.
- String과 메모리 누수의 관계를 설명할 수 있다.
- static과 main의 실행 구조를 JVM 메모리 관점에서 설명할 수 있다.

---

## 🧭 앞으로의 흐름

이 문제를 해결하기 위해 다음 순서로 학습한다.

1. JVM 구조와 실행 과정 정리
2. Runtime Data Area 메모리 구조 이해
3. 객체 생성과 참조 흐름 추적
4. GC의 실제 동작 실험
5. 메모리 실험 결과 분석
6. 면접 답변 형태로 정리

> **"왜 이 문제가 발생했는지 JVM 관점에서 완전히 설명할 수 있게 되는 것"**이다.
