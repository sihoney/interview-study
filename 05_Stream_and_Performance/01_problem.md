# 01. Stream & Performance — Problem Definition

---

## 🧨 장애 시나리오

대용량 로그 데이터를 집계하는 서비스에서  
처리 속도가 급격히 느려지고,  
CPU는 충분한데 처리량이 나오지 않는다.

기존의 for-loop 기반 처리는 코드가 길고 복잡하며  
병렬 처리 적용도 어렵다.

---

## 🔎 관찰된 증상

- 데이터 처리 지연
- CPU 활용 비효율
- 코드 가독성 저하
- 병렬 처리 구조 부재

---

## ❓ 핵심 질문

1. Stream은 왜 등장했는가?
2. 내부 반복은 어떤 장점이 있는가?
3. 병렬 스트림은 언제 사용해야 하는가?
4. ForkJoinPool과 어떤 관계가 있는가?

---

## 🧠 학습 목표

이 과정을 통해 다음을 설명할 수 있어야 한다.

- Stream API 설계 목적
- Lazy Evaluation의 의미
- 병렬 스트림의 구조와 성능 특성
- 대용량 데이터 처리 설계 전략
