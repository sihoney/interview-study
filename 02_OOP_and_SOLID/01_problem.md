# 02. OOP & SOLID — Problem Definition

---

## 🧨 프로젝트 상황

다음과 같은 문제가 발생했다.

> 기능이 늘어날수록 코드 한 파일이 수천 줄이 된다.  
> 기능 하나 수정하면 다른 기능이 연쇄적으로 깨진다.  
> 새로운 기능 추가 시 기존 코드를 계속 수정해야 한다.

---

## 🔎 관찰된 증상

- 파일 크기 급증
- 중복 코드 폭증
- 수정 시 버그 발생
- 테스트 불가능 구조
- 기능 추가가 공포

---

## ❓ 핵심 질문

1. 왜 코드가 이렇게 무너지는가?
2. 기능 중심 설계의 한계는 무엇인가?
3. 객체지향 설계는 이 문제를 어떻게 해결하는가?
4. SOLID 원칙은 어떤 역할을 하는가?
5. 추상 클래스와 인터페이스는 왜 필요한가?

---

## 🧠 학습 목표

이 과정을 통해 다음을 설명할 수 있어야 한다.

- 객체지향의 본질
- SOLID 원칙의 실제 의미
- 확장 가능한 설계 구조
- 인터페이스 기반 시스템 설계
- 예외 설계 전략

---

## 🧭 앞으로의 흐름

1. 객체지향 이론 정리
2. 절차형 코드 분석
3. 객체지향 설계 리팩토링
4. SOLID 적용 실습
5. 설계 전후 비교 분석
6. 면접 답변 정리

---

> 이 단계의 목적은  
> **"코드가 왜 무너지고, 어떻게 살릴 수 있는지"를 설계 관점에서 완전히 이해하는 것**이다.
