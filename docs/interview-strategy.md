# Interview Strategy

## 면접 답변 공식

항상 다음 순서로 답변한다.

1. 문제 상황 제시
2. 원인 분석
3. 설계 판단
4. 구현 방법
5. 결과 및 개선 효과

---

## 예시 답변 구조

"서비스 장애가 발생했을 때 GC 로그를 분석해 보니  
Full GC가 과도하게 발생하고 있었고, 이는 Stop-The-World로 이어져  
시스템 전체가 멈춘 것처럼 보였습니다.  
그래서 객체 생성 패턴을 수정하고 String 사용 방식을 개선했으며  
GC 빈도와 응답 시간이 크게 안정화되었습니다."

---

## 합격 시그널

- 장애 상황을 시스템적으로 설명한다.
- 코드 구조를 설계 관점에서 말한다.
- 성능 문제를 JVM과 연결해서 설명한다.
